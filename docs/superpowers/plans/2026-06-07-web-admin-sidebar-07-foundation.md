# Web Admin Sidebar 07 Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Adapt shadcn `sidebar-07` into the `web-admin` app shell so future pages render inside a ready collapsible-sidebar foundation.

**Architecture:** Generate the missing shadcn primitives into the existing `apps/web-admin/src/shared/ui` surface, then add app-owned navigation metadata that is passed down into shared shell components. `SidebarInset` becomes the only `main` landmark, pages keep data behavior, and the shell owns sidebar state, breadcrumbs, theme placement, and template-native user/team placeholders.

**Tech Stack:** Bun workspaces, Nx 20, Vite 5, React 19, React Router 7, TypeScript 5.5, Tailwind CSS 4, shadcn/ui Radix Rhea, Radix primitives through `radix-ui`, lucide-react, Vitest/jsdom, Testing Library, Playwright, GRACE XML.

- Official shadcn references:
  - `https://ui.shadcn.com/blocks#sidebar-07`
  - `https://ui.shadcn.com/docs/components/sidebar`
  - `https://ui.shadcn.com/r/styles/radix-rhea/sidebar-07.json`
- Approved decisions:
  - Adapt the full `sidebar-07` behavior, including desktop icon collapse.
  - Use template-native labels and routes; do not ship `Acme` or other shadcn demo content.
  - Preserve the existing page rule: admin pages import UI only from bare `@shared/ui`.
  - Keep route metadata app-owned and pass it into shared shell components through props.
  - `SidebarInset` is the only `main` landmark after migration.
  - Add sidebar theme tokens and token contract coverage.
  - Prove collapsed icon-rail behavior in e2e, not only trigger clickability.
  - Cover new handwritten UI-kit source without broad UI-kit coverage exclusions.

## Scope Check

This is one implementation plan. The primitive generation, tokens, shell components, route metadata, page migration, tests, e2e, and GRACE docs are tightly coupled: the foundation is only correct when the shared shell, app-owned navigation data, route rendering, and verification contract agree.

This plan does not add real authentication, real user profile loading, product dashboards, or production deployment for the Vite admin app.

      },

      },
    ],
  },

  initials: 'DV',

function normalizePathname(pathname: string): string {
  const cleanPath = pathname.split('?')[0]?.split('#')[0] || '/';
  if (cleanPath.length > 1 && cleanPath.endsWith('/')) {
    return cleanPath.slice(0, -1);
  }
  return cleanPath;
}

function cloneItemWithActive(item: AdminNavigationItem, pathname: string): AdminNavigationItem {
  const isActive = item.match ? item.match(pathname) : item.href === pathname;
  return {
    ...item,
    isActive,
    children: item.children?.map((child) => ({
      ...child,
      isActive: !child.disabled && child.href === pathname,
    })),
  };
}

function buildBreadcrumbs(pathname: string): AdminBreadcrumbItem[] {
  if (pathname === '/users') {
    return [{ label: 'Users' }];
  }
  if (pathname.startsWith('/users/')) {
    return [{ label: 'Users', href: '/users' }, { label: 'User detail' }];
  }
  if (pathname === '/ui-kit') {
    return [{ label: 'UI Kit' }];
  }
  return [{ label: 'Overview' }];
}

// START_CONTRACT: resolveAdminShellState
//   PURPOSE: Derive shell navigation display state from a browser pathname without importing app data into shared UI.
//   INPUTS: { pathname: string - browser location pathname, optionally with search or hash }
//   OUTPUTS: { navigation, referenceItems, breadcrumbs, user, teams - shell props for AdminAppShell }
//   SIDE_EFFECTS: none.
//   LINKS: M-WEB-ADMIN / V-M-WEB-ADMIN.
// END_CONTRACT: resolveAdminShellState
export function resolveAdminShellState(pathname: string) {
  const normalizedPathname = normalizePathname(pathname);
  return {
    navigation: baseNavigationGroups.map((group) => ({
      ...group,
      items: group.items.map((item) => cloneItemWithActive(item, normalizedPathname)),
    })),
    referenceItems: adminReferenceItems,
    breadcrumbs: buildBreadcrumbs(normalizedPathname),
    user: adminShellUser,
    teams: adminShellTeams,
  };
}
