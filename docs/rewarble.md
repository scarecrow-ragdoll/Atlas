1. Обзор продукта
Проблема
Продавцы цифровых ваучеров и gift cards на маркетплейсах (Eneba, Kinguin, G2A) платят Rewarble и аналогам комиссию 7%+ за инфраструктуру редима. При GMV $300–500k/мес это $21–35k в месяц потерь. При этом они не контролируют UX, лимиты, поставщиков и правила платформы.
 
Решение
Redeem.io - B2B инфраструктура цифровых ваучеров. Партнёры получают пак ваучеров, продают через маркетплейсы. Конечный пользователь активирует код на сайте и мгновенно получает цифровой товар (Steam, Xbox, PSN и др.). Платформа зарабатывает на марже дистрибуции.
 
Ключевые принципы (MUST, не обсуждаются):
— NO WALLET / NO BALANCE — система не хранит пользовательский баланс
— NO PAYOUT / NO CASH-LIKE — нет выплат, выводов, банковских переводов
— FULL REDEEM ONLY — ваучер используется полностью, остаток всегда 0
— FINALITY AFTER DELIVERY — после выдачи кода возврат невозможен

3. Финансовая модель
Бизнес-модель: партнёр покупает ваучеры у платформы с дисконтом, продаёт на маркетплейсе с наценкой. Платформа зарабатывает на марже дистрибуции между закупкой у поставщика и ценой для партнёра.
 
Пример транзакции:
— Номинал ваучера: $100
— Партнёр платит платформе: $98
— Себестоимость цифрового товара у поставщика: $90
— Маржа платформы на транзакцию: $8 (8%)

8.2 Стилистика и UX паттерны
Параметр
Решение
Настроение
Надёжный, быстрый, профессиональный. Не gaming, умеренно яркий (доверие важнее эмоций).
Tone of voice (B2C)
Дружелюбно, коротко, без лишних слов. 'Введи код - выбери - получи'.
Tone of voice (B2B/Admin)
Деловой, точный, никаких лишних украшений.
Референс (нравится)
Rewarble - за простоту редим-флоу (надежное API). skine.com - полный список партнеров, Coinsbee - понятный каталог.
Антиреференс (НЕ хотим)
Старые e-wallet интерфейсы - ощущение финансовой системы.
Запрещённые паттерны
Любые визуальные сигналы 'кошелька': balance widget, номер счёта, сумма 'на балансе'. Термины:   balance, payout, withdrawal.
Платформы
Веб (MVP). Mobile-responsive обязателен для B2C (пользователи часто с телефона). B2B/Admin —   desktop-first.
Навигация B2C
Минимальная. Логотип + ссылка на /how-to-buy + ссылка на /redeem. Никакого sidebar.
Навигация B2B
Sidebar: Dashboard / Issue / Transactions / Settings
Пустые состояния
Иллюстрация (или простая SVG) + короткий текст + CTA
Загрузка
Скелетоны для каталога SKU. Spinner для fulfillment-ожидания с ETA.
Ошибки
Toast-уведомления. Inline-ошибки у полей. Отдельные страницы для критических ошибок.
