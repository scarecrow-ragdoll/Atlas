# Шаблоны файлов docs/product/

Используй эти шаблоны при генерации файлов. Заполняй только то, что подтвердил аналитик. Неизвестное помечай `[ТРЕБУЕТ УТОЧНЕНИЯ]`.

## vision.md

```md
# Видение продукта

## Проблема

{проблема из ответов аналитика}

## Решение

{решение из ответов аналитика}

## Ценностное предложение

{ценность из ответов аналитика}

## Целевая аудитория

| Сегмент   | Описание   | Размер                           |
| --------- | ---------- | -------------------------------- |
| {сегмент} | {описание} | {размер или [ТРЕБУЕТ УТОЧНЕНИЯ]} |

## Конкурентный ландшафт

| Конкурент   | Что делает | Наше отличие |
| ----------- | ---------- | ------------ |
| {конкурент} | {описание} | {отличие}    |

## Метрики успеха

**North Star:** {метрика}

| KPI   | Целевое значение | Как измеряем |
| ----- | ---------------- | ------------ |
| {kpi} | {значение}       | {способ}     |

## Ограничения и допущения

- {ограничение}
```

## index.md (обновление статуса)

При обновлении `index.md` после завершения этапа, меняй статус соответствующей секции на `complete`. Формат строки таблицы:

```
| {Секция} | [{файл}]({путь}) | complete |
```
