# /ba-jira — Экспорт в Jira

Ты экспортируешь продуктовую документацию из `docs/product/` в Jira через MCP.

## Режим

Определи по `$ARGUMENTS`:

- **Пустой** → Экспорт всех фич
- **FT-NNN** → Экспорт одной фичи
- **--dry-run** (в любом сочетании) → Показать план экспорта без создания тикетов
- **FT-NNN --dry-run** → Dry-run только для указанной фичи

## Процесс

### 1. Пререквизит: inline-проверка

Перед экспортом проверь:

- Нет маркеров `[ТРЕБУЕТ УТОЧНЕНИЯ]` в файлах фич
- У каждой экспортируемой фичи есть AC (Given/When/Then)
- У каждой фичи указаны доступы

Если найдены блокеры — откажи в экспорте:

> Экспорт невозможен. Найдены проблемы:
>
> - {список проблем}
>   Запусти `/ba-review` для полной проверки или исправь вручную.

### 2. Сбор данных

1. Прочитай `docs/product/features/index.md` — извлеки группы фич (темы/эпики) и таблицу фич
2. Прочитай каждый `docs/product/features/FT-NNN-*.md` — извлеки stories, AC, описание
3. Прочитай `docs/product/business-rules.md` — контекст для описания эпиков
4. Прочитай `docs/product/data-dictionary.md` — контекст для описания эпиков

### 3. Маппинг → Jira

| docs/product/                              | Jira                | Поля                                                                                    |
| ------------------------------------------ | ------------------- | --------------------------------------------------------------------------------------- |
| Группа фич (тема из `features/index.md`)   | **Epic**            | summary = название темы, description = бизнес-правила + данные                          |
| `FT-NNN-slug.md`                           | **Task** (под Epic) | summary = название фичи, description = описание + все AC чеклистом                      |
| Каждый блок «As a... I want... So that...» | **Sub-task**        | summary = краткое название story, description = story + соответствующие Given/When/Then |
| MoSCoW приоритет                           | **Priority**        | Must → Highest, Should → High, Could → Medium, Won't → Low                              |

### 4. Dry-run

Если указан `--dry-run`, выведи план без создания тикетов:

```
## План экспорта в Jira

### Epic: {название темы}
  - Task: FT-{NNN} — {название} [Priority: {приоритет}]
    - Sub-task: {US-1 название}
    - Sub-task: {US-2 название}
  - Task: FT-{NNN} — {название} [Priority: {приоритет}]
    ...

Всего: {N} Epic, {M} Task, {K} Sub-task

Создать тикеты? (да/нет)
```

Если аналитик отвечает «да» — перейди к шагу 5. Если «нет» — ответь: «Экспорт отменён. Запусти `/ba-jira` снова когда будешь готов.»

### 5. Создание тикетов

Найди доступные Jira MCP инструменты через ToolSearch (ищи по ключевым словам `jira`, `issue`, `create`). Используй найденные инструменты для создания тикетов в порядке:

1. Создай Epic
2. Создай Task под Epic
3. Создай Sub-task под Task

После создания выведи:

> Экспорт завершён:
>
> - {N} Epic создано
> - {M} Task создано
> - {K} Sub-task создано
>
> Ссылки: {список ссылок на созданные Epic}
